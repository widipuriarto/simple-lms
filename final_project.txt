1. Overview  
Final project ini merupakan kelanjutan dari penugasan semester sebelumnya. Mahasiswa diminta mengembangkan 
kembali project Simple Learning Management System (LMS) yang sudah dibuat pada tugas capstone, lalu 
menambahkan fitur lanjutan, peningkatan kualitas sistem, dan integrasi backend yang lebih kompleks sesuai minat 
dan kemampuan masing -masing.  
Mahasiswa tidak wajib mengerjakan semua fitur. Pilih beberapa fitur tambahan dari daftar yang disediakan untuk 
mengumpulkan poin. Disarankan memilih 3 –5 fitur besar yang benar -benar bisa diselesaikan dan diuji dengan baik.  
Tujuan  
• Mengintegrasikan seluruh materi mata kuliah Pemrograman Sisi Server.  
• Melatih mahasiswa mengembangkan backend yang lebih realistis.  
• Memberikan fleksibilitas agar mahasiswa bisa memilih fitur sesuai minat.  
• Mendorong eksplorasi fitur lanjutan: caching, async task, analytics, testing, deployment, security.  
• Menilai kemampuan problem solving, desain sistem, dokumentasi, dan implementasi backend.  
 
2. Basis Project Minimal  
Final project harus melanjutkan project Simple LMS dari tugas sebelumnya. Minimal project sudah memiliki:  
• Docker dan Docker Compose.  
• Django project dengan PostgreSQL.  
• Model utama: User, Category, Course, Lesson, Enrollment, Progress.  
• REST API menggunakan Django Ninja.  
• JWT authentication.  
• Role -based access control minimal (admin, instructor, student).  
• Endpoint dasar untuk course, enrollment, dan progress.  
• Dokumentasi API via Swagger/OpenAPI.  
Jika project sebelumnya belum lengkap, boleh diperbaiki terlebih dahulu. Namun, perbaikan fitur dasar tidak dihitung 
sebagai fitur tambahan final project.

8. Format Laporan Final Project  
Buat file FINAL_PROJECT_REPORT.md di root repository dengan struktur berikut:  
• Identitas: Nama, NIM, Kelas, URL Repository.  
• Deskripsi Project: Jelaskan singkat project LMS yang dikembangkan.  
• Fitur Dasar yang Sudah Berjalan: Daftar fitur dasar: Authentication, Course API, Enrollment, Progress, dll.  
• Fitur Tambahan yang Dipilih: Tabel berisi: No, Fitur, Kategori, Poin, Status (Selesai/Sebagian/Belum).  
• Penjelasan Implementasi: Jelaskan fitur tambahan utama yang dibuat.  
• Cara Menjalankan Project: Langkah menjalankan dengan Docker Compose.  
• Akun Demo: Username/password untuk admin, instructor, student.  
• Endpoint Penting: Daftar endpoint yang perlu diuji.  
• Screenshot / Bukti Pengujian: Screenshot Swagger, Postman, atau hasil test.  
• Kendala dan Solusi: Kendala teknis yang dihadapi dan cara mengatasinya.  
• Kesimpulan: Refleksi singkat tentang pengalaman mengerjakan final project.
